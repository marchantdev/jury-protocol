## Judging Criteria

Colosseum explicitly describes this as "an intensive engineering AND business sprint." From their guide "How to Win a Colosseum Hackathon":

- **Impact** — The Grand Champion goes to "the most impactful product submission"
- **Technical Execution** — Working code, not mockups. On-chain programs preferred over off-chain scripts
- **Business Viability** — Accelerator evaluation looks at founder quality, market opportunity, business model
- **Differentiation** — Among 1,400+ submissions, what makes THIS memorable?
- **Completion** — A working product beats a grand vision. Ship beats dream.

Historical pattern from winners: Infrastructure primitives (TAPEDRIVE, Txtx), consumer products with unique insight (Unruggable, TypeX), DeFi innovations (Reflect, Urani, MCPay). Grand Prize goes to projects that combine technical depth with clear market potential.

Based on previous Colosseum hackathons and the Frontier arena page:

- **One-line description** — single sentence pitch
- **Category** — select from inspiration categories (Agents, DeFi, Payments, Privacy, etc.)

- **GitHub Repository URL** — public repo with code created during hackathon
- **Pitch Video URL** — YouTube/Loom/Vimeo link (~3 min)
- **Technical Demo URL** — separate video showing working product (2-3 min)
- **Live demo URL** (optional but strongly recommended)
- **Project description** — detailed writeup
- **Tech stack** — what Solana primitives you used

**Gotchas from experience:**
- Video links MUST be accessible (not private/unlisted without access)
- GitHub repo must show hackathon-period commits (judges check git log timestamps)
- Deployed URL significantly increases scoring (OWS hackathon taught us this)
- README quality matters — judges skim many projects. Clear README = more attention.

## Known SDK/Platform Bugs and Rough Edges

1. **Confidential Balances JS SDK gap**: No JavaScript/TypeScript library for ZK proof generation exists yet. Must use Rust backend or Wallets-as-a-Service. This is the BIGGEST developer friction for CB adoption.
2. **Token-2022 compatibility**: Not all wallets display Token-2022 tokens correctly. Phantom supports them; some others may not show balances or metadata.
3. **Anchor + Token-2022**: Anchor's `token` module defaults to SPL Token (not Token-2022). Must explicitly use `token_2022` interfaces when working with Token Extensions.
4. **Devnet vs Mainnet**: Some features (especially CB) may have subtle behavior differences. Test on devnet but note mainnet readiness in submission.
5. **ZK Compression state**: Compressed state is not yet universally indexed by all RPC providers. Helius RPCs support it; others may not.
6. **Video hosting**: Loom links expire after 30 days on free tier. Use YouTube (unlisted) for permanent links.
7. **Git timestamp verification**: Colosseum judges have tools to verify commit timestamps. Do NOT backdate commits or force-push to alter history.
